# 방사형 차트 정규화
- 비교 페이지 레이더(방사형) 차트의 데이터 왜곡 문제를 해결

---

## 1. 개요 및 변경 목적

### 기존 방식
- 선택된 3개 지역 간의 상대 평가 (Min-Max)
- 최하위 지역이 무조건 0점으로 처리되어 차트 왜곡 발생

### 개선 방향
- **전국 단위 고정 척도(절대적 위치)**와 **동일 권역 내 상대 평가(상대적 위치)**를 동시 제공.

---

## 2. 핵심 정규화 산식 (백분위 순위)

한국 지역 데이터의 특성상 특정 지역(예: 강남구)의 소비액이나 자산 가치가 비정상적으로 높은 **롱테일(Long-tail) 분포**를 보임.
이에 단순 선형 변환(Min-Max) 대신 분포의 치우침에 영향을 받지 않는 **백분위 산식**을 적용하고자 함.

### [백분위 점수 계산 공식]

```
점수 = (나보다 작은 지역 수 + 0.5 × 나와 같은 값인 지역 수) ÷ 전체 지역 수(N) × 100
```

---

## 3. 권역 구분 기준 (통계청 표준 6대 권역)

전국 250개 시군구를 행정구역 및 지리적 인접성에 따라 **통계청 표준 6대 권역**으로 분류
두 번째 차트인 '권역 내 차트'는 사용자가 선택한 지역들이 속한 아래의 권역 시스템을 기준으로 작동

| 권역명 | 포함 행정구역 (시·도 단위) |
| :--- | :--- |
| **수도권** | 서울특별시, 인천광역시, 경기도 |
| **강원권** | 강원특별자치도 |
| **충청권** | 대전광역시, 세종특별자치시, 충청북도, 충청남도 |
| **호남권** | 광주광역시, 전북특별자치도, 전라남도 |
| **영남권** | 부산광역시, 대구광역시, 울산광역시, 경상북도, 경상남도 |
| **제주권** | 제주특별자치도 |

---

## 4. 권역별 매칭 시나리오

사용자가 비교하기 위해 선택하는 3개 지역의 조합에 따라, 시스템은 자동으로 권역을 판별하여 차트 2(오른쪽 차트)의 기준을 정의하거나 예외 처리를 수행

### 시나리오 1: 동일 광역 지자체 내 비교 (예: 강남구, 종로구, 마포구)
- **판별:** 3개 지역의 상위 시·도가 모두 동일함 (`서울특별시`)
- 차트 2의 기준을 **"서울특별시 내"**로 고정하여 25개 자치구 간의 순위를 비교

### 시나리오 2: 동일 권역 내 타 광역 비교 (예: 서울 강남구, 인천 부평구, 경기 수원시)
- **판별:** 시·도는 다르나, 통계청 표준 권역상 모두 `수도권`에 포함됨.
- 차트 2의 기준을 **"수도권 권역 내"**로 설정하여 수도권 전체 시군구 기준 백분위를 재계산 후 표출.

### 시나리오 3: 서로 다른 권역 간 비교 (예: 서울 강남구, 부산 해운대구, 제주 제주시)
- **판별:** 선택된 지역들의 권역이 `수도권`, `영남권`, `제주권`으로 상호 상이함
- 차트 2 영역에 **"권역 비교 불가"** 안내 (→ 굳이 묶지 않아도 될 데이터를 묶어서 정보 왜곡이 생길 수 있어서.. 뺐습니다)
